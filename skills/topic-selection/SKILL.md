# topic-selection

## 目的

判断一个选题值不值得写。

核心不是问「这个热点火不火」，而是问：

> 这个选题能不能回答一个好问题。

## 使用场景

```text
看到热点新闻
看到别人的爆款观点
准备写公众号
准备做短视频
判断一个赚钱机会
```

## 判断流程

### 1. 这个选题背后真正的问题是什么

不要停在标题。

比如：

```text
AI 又更新了
```

这不是问题。

真正的问题可能是：

```text
普通人为什么越追 AI 工具越焦虑？
```

### 2. 它和用户有什么关系

必须能落到普通人的生活里。

```text
省钱
赚钱
少踩坑
少焦虑
提高效率
重新拿回控制权
```

如果只能写成行业分析，普通人不关心，就先放弃。

### 3. 有没有个人切口

优先从用户真实经历进入：

```text
送快递
写公众号
戒烟
熬夜
炒币失败
手机用 AI
中午吃盒饭看新闻
```

没有个人切口，文章容易变成搬运。

### 4. 有没有反常识

好选题通常有一个反常识点：

```text
你以为是机会，其实是诱饵
你以为是工具不够，其实是行动不够
你以为是自制力差，其实是环境一直在拉你
```

### 5. 是否容易踩事实坑

涉及新闻、产品发布时间、公司动作、法律政策、价格、模型能力时，必须查证。

不要从标题推断「最近」。

## 输出格式

```text
选题判断：值得写 / 暂缓 / 不建议写

核心问题：……

普通人关系：……

个人切口：……

风险点：……

建议标题：……
```

## 禁止

```text
因为热点火就建议写
把所有新闻都升华成人性
没有查证就说新功能、新产品、新趋势
```
