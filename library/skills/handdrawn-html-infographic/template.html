循环的四个部分

TRIGGER时间表或新笔记保存
→
DO读取知识库并完成工作
→
VERIFY对照硬规则检查
→
STOP通过或达到重试上限

跳过 VERIFY

模型会给自己的作业打高分，却没有真正验证结果。

跳过 STOP

任务可能持续运行，悄悄增加 API 费用。

模块一

  1. 读取原始材料
  2. 提取决定、推理和后续结果
  3. 一条笔记只保留一个想法
  4. 发现重复主题时更新旧笔记
在这里放 VERIFY 和 STOP 条件。

模块二

轮次 1

搁置线索

轮次 2

什么有效

轮次 3

诚实审视

轮次 4

隐藏重复

真正可行的顺序

  1. 先手动运行
  2. 再拆成两个循环
  3. 先加 VERIFY 和 STOP
  4. 最后才安排自动运行

先让手动版本可靠，再让自动化替你长期重读。
