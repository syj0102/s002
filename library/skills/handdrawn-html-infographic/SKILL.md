# Handdrawn HTML Infographic Skill

## 目标

把用户提供的长文本，转成一份可直接打开、可复制文字、可截图、可继续修改的手绘风 HTML 信息图。

最终输出必须是完整 `.html` 文件，而不是图片。

## 适用场景

- 手绘风知识卡片
- 流程图
- 教程海报
- 长文章视觉化
- 公众号配图
- 手机浏览器直接查看

## 核心原则

1. 所有中文必须使用真实 HTML 文本，不把中文写进生成图片。
2. 图标只用 emoji、CSS 图形或内联 SVG。
3. 页面必须适配手机，宽度不超过 1080px。
4. 视觉风格要像手绘笔记，不要像企业 PPT。
5. 不追求把全文塞进一张图，先提炼结构，再保留关键原句。
6. 重要警告、规则、结论必须有明显视觉区分。
7. 用户原文中的关键术语、流程名、数字、条件不能擅自改写。

## 处理流程

### 第一步 识别信息结构

从原文中提取：

- 一个核心标题
- 一段开场说明
- 1 个主流程
- 2 到 4 个分区
- 关键警告
- 示例
- 最后建议

### 第二步 压缩内容

压缩规则：

- 每个卡片 40 到 120 字
- 每个列表 3 到 6 条
- 每条尽量不超过两行
- 保留数字、时间、触发条件、停止条件
- 删除重复解释
- 不编造原文没有的例子

### 第三步 生成 HTML

必须输出以下结构：

- 页面标题区
- 对比区
- 主流程区
- 两个循环或模块区
- 警告区
- 实施顺序区
- 成本或注意事项区
- 最后建议区

### 第四步 自检

输出前检查：

- 中文是否全部为真实文本
- 是否存在乱码字符
- 是否有未闭合标签
- 手机宽度是否正常
- 是否有横向溢出
- 标题是否清楚
- 原文关键规则是否保留
- 是否遗漏 VERIFY 和 STOP

## HTML 技术要求

- 使用 UTF-8
- 使用单文件 HTML
- CSS 写在 `<style>` 内
- 不依赖外部 JS
- 不依赖外部图片
- 可以使用内联 SVG
- 字体优先：
  - `KaiTi`
  - `STKaiti`
  - `FZKai-Z03`
  - `Noto Serif SC`
  - `serif`
- 正文字体可搭配：
  - `Microsoft YaHei`
  - `PingFang SC`
  - `sans-serif`

## 推荐视觉风格

颜色：

- 纸张背景 `#F8F1DF`
- 墨色 `#1E1E1E`
- 蓝色 `#2563EB`
- 绿色 `#16A34A`
- 黄色 `#FACC15`
- 橙色 `#F97316`
- 红色 `#DC2626`

元素：

- 卡片轻微旋转
- 手绘边框
- 不规则下划线
- 虚线箭头
- 便签纸
- 圆圈数字
- 警告三角形
- 云朵气泡

## 输出格式

完成后：

1. 创建一个 `.html` 文件。
2. 文件名使用中文主题或英文简写。
3. 确认文件能直接在浏览器打开。
4. 向用户提供下载链接。
5. 不只贴代码，必须生成真实文件。
